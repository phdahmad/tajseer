# نتائج الاختبار النصفي - برمجة الحاسب CS1021 (الشعبة 47)

موقع ويب بسيط يسمح للطلبة بتسجيل الدخول بالرقم الجامعي ومشاهدة:
- **الدرجة النهائية والتقدير** مع رسم دائري
- **توزيع الدرجة** على اقسام الاختبار (اختيار من متعدد، صح/خطأ، اجابات قصيرة)
- **التصحيح التفصيلي** لكل سؤال مع الاجابة الصحيحة والملاحظة
- **ملاحظات وتوصيات** خاصة بالطالب
- **روابط لتحميل التقرير** بصيغة PDF و Word، وكذلك ورقة الاجابة الاصلية

## محتويات المجلد

```
website/
├── index.html           # الصفحة الوحيدة (تسجيل دخول + عرض النتائج)
├── README.md            # هذا الملف
└── reports/             # ملفات الطلبة (مفهرسة بالرقم الجامعي)
    ├── 447055077_report.pdf      تقرير التصحيح PDF
    ├── 447055077_report.docx     تقرير التصحيح Word
    ├── 447055077_answer.pdf      ورقة الاجابة المسحوبة
    ├── ...
    └── (نفس الملفات لكل طالب)
```

## كيف يستخدمها الطلبة

1. يفتح الطالب رابط الموقع
2. يدخل رقمه الجامعي (مثل `447055100`)
3. يضغط زر "عرض النتيجة"
4. يشوف نتيجته وتفاصيل اختباره
5. يقدر يحمل تقريره الكامل بـ PDF او Word

## النشر على GitHub Pages

### الخطوة 1: انشاء مستودع جديد على GitHub
- ادخل على github.com وسجل دخول
- اضغط زر **New repository**
- اختر اسم مثل: `cs1021-results` (لا تستخدم العربي في اسم المستودع)
- اجعله **Public** (لان GitHub Pages المجاني يحتاج Public)
- اضغط **Create repository**

### الخطوة 2: رفع الملفات
**الطريقة السهلة (عبر المتصفح):**
1. في صفحة المستودع، اضغط **Add file → Upload files**
2. اسحب ملف `index.html` ومجلد `reports/` بالكامل
3. اكتب رسالة commit مثل: "اضافة نتائج الاختبار النصفي"
4. اضغط **Commit changes**

**الطريقة المتقدمة (عبر Git command line):**
```bash
cd website
git init
git add .
git commit -m "اضافة نتائج الاختبار النصفي"
git branch -M main
git remote add origin https://github.com/USERNAME/cs1021-results.git
git push -u origin main
```

### الخطوة 3: تفعيل GitHub Pages
1. ادخل **Settings** الخاص بالمستودع
2. من القائمة اليسرى اضغط **Pages**
3. تحت **Source** اختر **Deploy from a branch**
4. اختر **Branch: main** و **Folder: / (root)**
5. اضغط **Save**
6. انتظر دقيقة او دقيقتين

### الخطوة 4: مشاركة الرابط مع الطلبة
رابطك سيكون:
```
https://USERNAME.github.io/cs1021-results/
```
ارسل هذا الرابط للطلبة (واتساب، ايميل، نظام الجامعة).

## ملاحظات امان

- الموقع لا يحفظ اي شيء على السيرفر، كل البيانات داخل ملف HTML واحد فقط
- اي شخص يعرف رقم جامعي صحيح يقدر يشوف نتيجة صاحبه - اذا تبي حماية اقوى راجع المالك
- لمنع الفهرسة من جوجل: اضف ملف `robots.txt` فيه:
  ```
  User-agent: *
  Disallow: /
  ```

## لوحة تحكم المدرس (Admin Dashboard)

**الرابط:** `https://USERNAME.github.io/cs1021-results/admin.html`

**كلمة السر:** `تجسير`

**ما تشوفه:**
- احصائيات شاملة (عدد الطلبة، المتوسط، اعلى وادنى درجة، الناجحين والراسبين)
- رسم بياني لتوزيع التقديرات
- جدول كل الطلبة مرتب حسب الدرجة (مع البحث، الفلترة، الترتيب)
- زر عرض تفاصيل لكل طالب (يفتح نافذة فيها كل التفاصيل)
- ازرار تحميل لكل ملف على حدة
- زر تحميل **كل التقارير** دفعة واحدة في ملف ZIP
- رابط للانتقال لصفحة الطلبة العادية

**ملاحظة امان:** كلمة السر مكتوبة في الصفحة (نظراً لانه نشر static)، لكن صفحة admin.html منفصلة وعنوانها غير معروف للطلبة. تقدر تغير كلمة السر بفتح ملف `admin.html` وتعديل السطر `const PASSWORD = "تجسير";`

## ميزة تتبع تسجيلات الدخول

تم ربط الموقع بـ Google Sheets عبر Apps Script لتسجيل كل محاولة دخول وعرضها في الداش بورد.

**ما يتم تسجيله:**
- الرقم الجامعي والاسم
- التاريخ والوقت تلقائياً
- نوع الجهاز (موبايل/تابلت/كمبيوتر)
- نوع المتصفح (Chrome/Safari/Firefox/Edge)
- محاولات الدخول الفاشلة (الارقام الخاطئة)

**ما يظهر في الداش بورد (قسم "نشاط الدخول"):**
- بطاقات احصائية: اجمالي الدخولات، عدد الطلبة الذين دخلوا، الذين لم يدخلوا، والمحاولات الفاشلة
- رسم بياني للدخولات حسب اليوم
- جدول كل طالب: عدد المرات، اول واخر دخول، الجهاز والمتصفح
- قائمة الطلبة الذين لم يدخلوا بعد (للتذكير)
- جدول محاولات الدخول الفاشلة (لاكتشاف الفضوليين)
- سجل كامل بكل التسجيلات بترتيب زمني
- زر "تحديث البيانات" لجلب اخر التسجيلات

**اين تخزن البيانات:**
في Google Sheet اسمه `CS1021_Login_Tracker` على Google Drive الخاص بك. تقدر تفتحه مباشرة وتشوف البيانات بدون الداش بورد.
